# XUANOS 页面线框说明 v0.1

- 文档版本：v0.1
- 当前状态：阶段 1 页面结构规格
- 适用版本：XUANOS MVP
- 页面数量：5
- 视觉方向：黑金东方极简、Apple 风格毛玻璃、低密度粒子、未来感系统文字
- 核心目标：定义每个页面的结构、模块、信息优先级和交互行为

---

# 1. 页面总览

XUANOS MVP 只做 5 个核心页面：

```text
1. 首页 / 控制台 / 对话入口
2. 引导式理解页
3. 计划裁决页
4. 行动反馈页
5. 我的系统页
```

核心流转：

```text
首页
→ 引导式理解页
→ 计划裁决页
→ 行动反馈页
→ 我的系统页
→ 回到首页或继续任务
```

页面原则：

> 每个页面只解决一个核心问题，不堆功能。

---

# 2. 全局布局

桌面端使用三栏布局：

```text
┌──────────────────────────────────────────────┐
│ 顶部状态栏                                    │
├──────────────┬──────────────────┬────────────┤
│ 左侧任务栏    │ 中间主工作区       │ 右侧系统摘要 │
└──────────────┴──────────────────┴────────────┘
```

## 左侧任务栏

包含：

- XUANOS 标识；
- 当前任务线程；
- 我的系统；
- 计划裁决；
- 现实边界；
- 行动执行；
- 复盘记录；
- 系统设置。

MVP 可以只让部分入口可点击。

## 中间主工作区

显示当前页面核心内容。

## 右侧系统摘要

展示：

- 当前主目标；
- 当前唯一行动；
- 系统仍在验证；
- 最近修正。

小屏幕下可以折叠。

# 3. 页面一：首页 / 控制台 / 对话入口

## 3.1 页面目的

首页要让用户立刻知道：

- 当前在处理什么；
- 今天最重要一步是什么；
- 可以继续上次任务；
- 可以新建目标；
- 可以自己说；
- 可以让 XUANOS 问。

首页不是普通聊天页，而是个人成长控制台。

---

## 3.2 页面结构

```text
┌──────────────────────────────────────────────┐
│ XUANOS                                       │
│ 你的成长，由系统校准                          │
├──────────────┬───────────────────────────────┤
│ 左侧任务线程  │ 主区域                         │
│              │                               │
│              │  当前主线卡片                   │
│              │  当前唯一行动卡片               │
│              │  表达方式选择                   │
│              │  聊天输入框                     │
│              │                               │
└──────────────┴───────────────────────────────┘
```

---

## 3.3 必须模块

### A. 品牌开场区

显示：

```text
XUANOS
个人成长操作系统
PERSONAL GROWTH OS
```

短句：

> 校准你的下一步。

不要写长篇欢迎语。

---

### B. 当前主线卡片

标题：

```text
当前主线
CURRENT VECTOR
```

字段：

```text
主目标：
当前阶段：
系统状态：
```

示例：

> 主目标：完成 XUANOS 静态前端原型  
> 当前阶段：视觉系统确认  
> 系统状态：等待行动

---

### C. 当前唯一行动卡片

标题：

```text
当前唯一行动
NEXT ACTION
```

字段：

```text
行动：
完成标准：
预计负荷：
```

示例：

> 行动：完成 UI_STYLE_GUIDE.md  
> 完成标准：可交给 Codex 生成静态原型  
> 预计负荷：中等

---

### D. 表达方式选择

三个按钮：

```text
我先完整表达
XUANOS 来问我
一起梳理
```

说明：

- 我先完整表达：用户自由输入；
- XUANOS 来问我：系统逐步提问；
- 一起梳理：用户先说一句，系统辅助整理。

---

### E. 指令输入框

placeholder：

> 输入你的目标、计划、困境，或让 XUANOS 开始提问。

加号菜单：

```text
新建目标
修改当前计划
提交行动结果
切换导师风格
上传已有计划
```

---

## 3.4 页面视觉

首页可以使用：

- 金色低密度粒子；
- 圆相或山水线稿；
- Apple-like 深色毛玻璃主卡片；
- 黑金按钮；
- 少量英文小标签。

首页不能像普通 AI 聊天框。

# 4. 页面二：引导式理解页

## 4.1 页面目的

让用户感到：

> XUANOS 正在逐步理解我，而不是直接套模板。

---

## 4.2 页面结构

```text
┌──────────────────────────────────────────────┐
│ 引导式理解                                    │
│ GUIDED UNDERSTANDING                         │
├──────────────────────────────────────────────┤
│ 阶段指示器                                    │
│ 系统理解摘要卡片                              │
│ 下一关键问题卡片                              │
│ 用户回答区                                    │
│ 用户纠正入口                                  │
└──────────────────────────────────────────────┘
```

---

## 4.3 阶段指示器

阶段：

```text
理解目标
核对现实
判断冲突
生成起点档案
准备计划
```

当前阶段使用金色高亮。

---

## 4.4 系统理解摘要

标题：

```text
系统理解
SYSTEM READ
```

格式：

```text
真实目标：
当前基础：
现实限制：
主要矛盾：
仍不确定：
```

示例：

> 真实目标：不是“学完 Python”，而是完成一个可运行小程序。  
> 主要矛盾：目标结果不清，无法判断学习路径。

---

## 4.5 下一关键问题

标题：

```text
下一问题
NEXT QUESTION
```

一次只问一个问题。

示例：

> 30 天后，你希望自己实际做出什么？  
> A. 看懂基础代码  
> B. 独立写小程序  
> C. 完成一个可以使用的应用

---

## 4.6 用户纠正入口

按钮：

```text
准确
部分准确
不准确
我要补充
```

若用户点击“不准确”，显示输入框：

> 哪一部分不准确？

---

## 4.7 页面视觉

引导页要有“系统扫描信息”的感觉，但不能像审问。

允许：

- 阶段线；
- 金色节点；
- Apple-like 毛玻璃理解卡；
- 少量粒子；
- 标签化内容。

禁止：

- 大段解释；
- 一次十几个问题；
- 客服聊天气质。

---

# 5. 页面三：计划裁决页

## 5.1 页面目的

计划页展示 XUANOS 的核心价值：

> 不只是安排任务，而是帮用户取舍。

---

## 5.2 页面结构

```text
┌──────────────────────────────────────────────┐
│ 计划裁决                                      │
│ PLAN DECISION                                │
├──────────────────────────────────────────────┤
│ 主目标卡片                                    │
│ 维持目标 / 暂停目标 / 删除事项                │
│ 今日时段计划                                  │
│ 当前唯一行动                                  │
│ 完成标准                                      │
│ 复查条件                                      │
│ 修改计划 / 接受计划                           │
└──────────────────────────────────────────────┘
```

---

## 5.3 主目标卡片

标题：

```text
主目标
PRIMARY VECTOR
```

字段：

```text
主目标：
裁决原因：
当前阶段：
主要风险：
```

示例：

> 主目标：完成 XUANOS 静态前端原型  
> 裁决原因：它是验证第一印象和产品气质的必要前置。  
> 主要风险：继续讨论理念，推迟真实开发。

---

## 5.4 目标取舍区

三列：

```text
维持目标
暂停目标
删除事项
```

示例：

维持目标：

- 每周基础健身 3 次

暂停目标：

- Flutter 客户端
- 完整商业系统
- 复杂登录注册

删除事项：

- 无意义动效
- 排行榜
- 积分系统

这部分必须明显体现“取舍”。

---

## 5.5 今日时段计划

标题：

```text
今日时段
TIME BLOCKS
```

默认不具体到分钟。

字段：

```text
高能时段：
普通时段：
低能时段：
```

示例：

> 高能时段：完成核心设计文档  
> 普通时段：整理页面线框  
> 低能时段：复查并修改文案

---

## 5.6 当前唯一行动

标题：

```text
当前唯一行动
NEXT ACTION
```

必须视觉突出。

字段：

```text
行动：
完成标准：
预计负荷：
```

示例：

> 行动：完成 PAGE_WIREFRAMES.md 第一版  
> 完成标准：Codex 能根据它生成五个静态页面  
> 预计负荷：中等

---

## 5.7 计划操作

按钮：

```text
接受计划
修改计划
我不同意这个判断
重新问我
```

用户修改后显示影响提示。

---

## 5.8 页面视觉

计划页必须有裁决感。

允许：

- 金色主目标卡；
- 暗金分割线；
- Apple-like 毛玻璃计划面板；
- 低密度粒子；
- 少量东方山水线稿。

禁止：

- 像待办清单；
- 平铺很多任务；
- 一屏大段解释。

---

# 6. 页面四：行动反馈页

## 6.1 页面目的

轻量收集真实结果，让系统修正判断。

---

## 6.2 页面结构

```text
┌──────────────────────────────────────────────┐
│ 行动执行                                      │
│ ACTION EXECUTION                             │
├──────────────────────────────────────────────┤
│ 原计划摘要                                    │
│ 是否开始                                      │
│ 是否完成                                      │
│ 实际用时                                      │
│ 最大阻力                                      │
│ 情绪 / 精力变化                               │
│ 原计划哪里不现实                              │
│ 提交反馈                                      │
│ 系统修正结果                                  │
└──────────────────────────────────────────────┘
```

---

## 6.3 必须字段

```text
是否开始
是否完成
完成比例
实际用时
最大阻力
情绪或精力变化
原计划哪里不现实
```

最大阻力选项：

```text
行动不明确
目标不认可
环境干扰
临时意外
他人影响
情绪变化
精力不足
能力不足
其他
```

---

## 6.4 提交后显示

标题：

```text
系统修正
SYSTEM REVISION
```

格式：

```text
原判断：
实际结果：
系统修正：
下一步调整：
```

示例：

> 原判断：你主要卡在目标过多。  
> 实际结果：目标减少后，你仍然没有开始，而是继续修改文档。  
> 系统修正：可能存在回避真人测试的倾向。  
> 下一步调整：把“完成测试”缩小为“联系一位同学约 15 分钟交流”。

---

## 6.5 页面视觉

反馈页要轻，不要像长篇复盘。

允许：

- 选择按钮；
- 进度条；
- 金色提交按钮；
- 提交后修正卡片淡入。

禁止：

- 大段输入；
- 复杂情绪量表；
- 长问卷。

---

# 7. 页面五：我的系统页

## 7.1 页面目的

让用户看到：

> XUANOS 正在形成一套属于我的个人系统。

它可以根据用户选择更像：

```text
日记本
任务本
记录本
个人成长档案
```

但展示必须简洁，不能暴露所有后台数据。

---

## 7.2 页面结构

```text
┌──────────────────────────────────────────────┐
│ 我的系统                                      │
│ PERSONAL SYSTEM                              │
├──────────────────────────────────────────────┤
│ 模式切换：档案模式 / 日记模式 / 混合模式       │
│ 当前主线                                      │
│ 现实边界                                      │
│ 对我有效                                      │
│ 系统仍在验证                                  │
│ 当前计划                                      │
│ 最近修正                                      │
│ 我纠正过系统的内容                            │
└──────────────────────────────────────────────┘
```

---

## 7.3 展示模式

### 档案模式

更结构化，突出：

- 当前主线；
- 现实边界；
- 对我有效；
- 系统仍在验证；
- 行为证据；
- 最近修正。

### 日记模式

更私人，突出：

- 最近状态；
- 最近发生了什么；
- 用户自己的表达；
- 系统温和复述；
- 行动记录。

### 混合模式

默认模式。

上半部分为系统档案，下半部分为近期日记记录。

# 10. 页面验收标准

阶段 1 原型必须满足：

1. 用户能从首页进入任一核心流程；
2. 首页不像普通 AI 聊天框；
3. 黑金风格统一；
4. 粒子和 Apple-like 毛玻璃体现系统感；
5. 东方极简明显但不玄学；
6. 引导页有清晰阶段感；
7. 计划页有明显取舍感；
8. 反馈页足够轻；
9. 我的系统页有档案 / 日记模式切换；
10. 用户能看到系统判断可以被修改；
11. 页面信息短句化、标签化；
12. Codex 后续能根据这些页面接入真实数据。
