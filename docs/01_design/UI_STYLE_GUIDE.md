# XUANOS UI Style Guide v0.1

- 文档版本：v0.1
- 当前状态：阶段 1 视觉规格
- 适用版本：XUANOS MVP
- 视觉方向：黑金东方极简、Apple 风格毛玻璃、低密度粒子、个人成长控制台
- 核心目标：定义 XUANOS 第一版前端的视觉气质、颜色、排版、组件风格和禁止事项

---

# 1. 视觉总原则

XUANOS 不是普通 AI 聊天网页，不是学习打卡软件，也不是企业后台。

它应该像一个：

> 深色、克制、黑金、东方极简、有系统感的个人成长控制台。

用户进入后应感到：

- 这里不是闲聊工具；
- 系统正在理解我；
- 我的目标、计划、行动和反馈会被持续记录；
- 这里有温度，但不会一味安慰；
- 这里会帮我面对现实，而不是制造更多幻想。

XUANOS 的视觉必须同时具备两种感觉：

1. **系统感**：像一个正在运行的个人成长操作系统；
2. **人味**：像一本属于用户自己的日记本、任务本、档案本。

用户可以在“我的系统”页中选择更偏：

- 档案模式；
- 日记模式；
- 混合模式。

# 3. 色彩系统

## 3.1 主背景

主背景必须是深黑色系，不使用浅色背景。

```text
主背景：#050505
次背景：#0A0A08
深层背景：#000000
暗金阴影背景：#120F08
```

视觉要求：

- 黑而不死；
- 有空间深度；
- 可以承载粒子和金色微光；
- 不得使用大面积纯灰。

---

## 3.2 卡片颜色

用于计划卡、系统摘要、个人档案、反馈卡片。

```text
主卡片：rgba(18, 16, 10, 0.58)
次卡片：rgba(28, 24, 14, 0.46)
悬浮卡片：rgba(35, 30, 18, 0.50)
卡片边框：rgba(191, 167, 106, 0.22)
弱边框：rgba(191, 167, 106, 0.10)
```

卡片必须是深色毛玻璃，不做浅色玻璃拟态。

---

## 3.3 金色系统

金色是 XUANOS 的核心强调色，但必须克制。

```text
主金色：#D6B76A
暗金色：#BFA76A
亮金色：#F3D98B
弱金色：rgba(214, 183, 106, 0.55)
极弱金：rgba(214, 183, 106, 0.18)
```

使用规则：

- 主按钮、关键状态、系统裁决可用主金色；
- 暗金用于边框、线条、分割、图标；
- 亮金只用于极少量焦点；
- 禁止大面积金色铺满页面；
- 金色必须像“系统微光”，不是“土豪金”。

---

## 3.4 文本颜色

```text
主文本：#F6EFD8
正文文本：#D8CDAE
次文本：#A99A72
弱文本：#6F6548
极弱文本：#4A422F
```

不要使用纯白。  
中文正文必须清楚、耐读。

---

## 3.5 状态色

即使整体黑金，也需要状态提示色，但必须低饱和。

```text
普通说明：#D6B76A
成功 / 已完成：#8FAF7A
影响提示：#D89B45
严重风险：#B75C4A
```

提示等级：

- 金色：普通系统提示；
- 橙金：用户选择可能影响目标；
- 暗红：安全风险或计划事实上不可行。

红色不能滥用。

---

# 4. 粒子与毛玻璃

## 4.1 粒子系统

XUANOS 允许使用低密度金色粒子背景。

粒子不是炫技，而是表达：

> 系统正在运行，个人信息正在被整理成结构。

粒子规则：

```text
桌面端：40-80 个粒子
移动端：15-30 个粒子
颜色：暗金、弱金
速度：极慢
透明度：低
连线：可以有，但必须极弱
```

允许效果：

- 星图感；
- 微弱金色节点；
- 缓慢漂浮；
- 极细线连接；
- 像东方星盘或系统经络。

禁止效果：

- 满屏快速粒子；
- 鼠标移动爆炸效果；
- 高亮霓虹网络；
- 游戏粒子；
- 遮挡正文；
- 影响性能。

用户开启减少动态效果时，应弱化或关闭粒子。

---

## 4.2 Apple 风格毛玻璃系统

XUANOS 的毛玻璃可以参考 Apple 的视觉语言：

- 透明但清晰；
- 模糊但不糊；
- 柔和、有层次；
- 光感克制；
- 交互时有轻微浮动和高亮。

但 XUANOS 不能做成浅色 iOS 风格。  
它必须是 **深色 Apple-like glass**。

推荐 CSS 方向：

```css
background: linear-gradient(
  135deg,
  rgba(30, 26, 16, 0.62),
  rgba(10, 10, 8, 0.42)
);
backdrop-filter: blur(22px) saturate(130%);
-webkit-backdrop-filter: blur(22px) saturate(130%);
border: 1px solid rgba(214, 183, 106, 0.18);
box-shadow:
  0 20px 60px rgba(0, 0, 0, 0.42),
  inset 0 1px 0 rgba(255, 255, 255, 0.05);
```

要求：

- 保证文字可读；
- 不要让每个元素都毛玻璃；
- 重点卡片使用毛玻璃；
- 普通模块可用深色实体卡片；
- 毛玻璃用于“系统面板”“档案面板”“计划裁决”“首页主卡片”；
- 滚动、hover、切换页面时可以有轻微 glass highlight。

禁止：

- 透明度过高导致文字读不清；
- 大面积亮白玻璃；
- 卡片边缘过亮；
- 全页面所有元素都套毛玻璃；
- 毛玻璃叠太多层导致廉价感。

# 5. 东方极简元素

允许使用：

```text
圆相
山水线稿
竹影
玄纹
星图
金色细线
留白
对称构图
竖向细分割线
```

禁止使用：

```text
大面积龙纹
符箓
神兽
强玄学图案
寺庙化视觉
复杂国潮插画
过度水墨扩散
```

XUANOS 要像“东方系统”，不是“玄学 App”。

# 6. 字体与排版

## 6.1 字体方向

中文标题要有一点未来感，但必须可读。  
正文使用现代无衬线，保证阅读舒适。

推荐字体栈：

```css
font-family:
  Inter,
  "SF Pro Display",
  "PingFang SC",
  "Microsoft YaHei",
  sans-serif;
```

后续可以替换为更有未来感的中文标题字体，但不得牺牲阅读。

---

## 6.2 字号规则

```text
页面大标题：28-36px
模块标题：20-24px
卡片标题：16-18px
正文：14-16px
辅助说明：12-13px
英文小标签：10-12px
```

## 6.3 英文小标签

页面标题可以使用中文主标题 + 英文小标签。

示例：

```text
我的系统
PERSONAL SYSTEM

计划裁决
PLAN DECISION

现实边界
REALITY BOUNDS

行动执行
ACTION EXECUTION
```

英文必须：

- 小；
- 淡；
- 字距略大；
- 不抢中文主信息。

---

# 7. 文案风格

界面文案必须短句化、标签化、状态化。

推荐表达：

```text
当前主线
目标载入
现实边界
计划负荷偏高
系统仍在验证
等待确认
冻结次要目标
当前唯一行动
已根据反馈修正
```

避免表达：

```text
根据你目前的各种情况，我认为你可能需要进行一些调整……
我非常理解你的感受，你其实已经很努力了……
下面我将为你制定一份详细完整的计划……
```

XUANOS 的语言像系统状态，不像聊天废话。

---

# 8. 页面布局原则

## 8.1 推荐布局

桌面端使用：

```text
左侧任务导航
中间主工作区
右侧个人系统摘要
```

小屏幕下右侧摘要折叠。

## 8.2 留白原则

XUANOS 要克制。

每个页面只突出一个核心问题：

- 首页：我现在要继续什么；
- 引导页：系统正在理解什么；
- 计划页：我现在做什么、不做什么；
- 反馈页：实际发生了什么；
- 我的系统页：系统目前如何理解我。

# 9. 组件风格

## 9.1 卡片 Card

```text
背景：深色 Apple-like 毛玻璃
圆角：18-28px
边框：弱金色
阴影：深黑柔和阴影
内边距：20-28px
```

卡片感觉应像“档案舱”或“系统面板”。

---

## 9.2 按钮 Button

### 主按钮

用于：

- 进入系统；
- 接受计划；
- 继续；
- 提交反馈。

样式：

```text
背景：金色渐变
文字：深黑
圆角：999px 或 14px
字体：中等偏粗
```

### 次按钮

用于：

- 修改计划；
- 查看依据；
- 重新问我。

样式：

```text
背景：深色透明
边框：弱金色
文字：金白色
```

### 风险按钮

用于：

- 删除目标；
- 不再使用此信息。

样式：

```text
暗红透明背景
弱红边框
浅红文字
```

---

## 9.3 输入框

输入框要像系统指令入口。

要求：

```text
深色毛玻璃
弱金边框
聚焦时金色边框增强
支持长文本
底部有快捷入口
```

placeholder：

> 输入你的目标、计划、困境，或让 XUANOS 开始提问。

---

## 9.4 标签 Tag

用于目标状态和系统判断。

示例：

```text
主目标
维持目标
暂停
待验证
已确认
用户修改
系统建议
计划负荷偏高
```

标签使用暗金低透明背景，不要太亮。

---

## 9.5 提示条 WarningBanner

等级：

```text
info：普通提示
impact：计划影响
risk：严重风险
```

示例：

> 该调整为用户最终选择，并非 XUANOS 当前首选建议。系统将在两次执行后复查。

# 10. “我的系统”展示模式

用户可在“我的系统”页选择信息展示风格。

## 10.1 档案模式

更像系统档案。

强调：

- 当前主线；
- 现实边界；
- 系统判断；
- 待验证假设；
- 行为证据；
- 最近修正。

适合喜欢理性分析、明确结构的用户。

## 10.2 日记模式

更像私人记录本。

强调：

- 最近状态；
- 今天发生了什么；
- 最近的目标变化；
- 用户自己的话；
- 系统温和复述；
- 行动记录。

适合更需要表达和情绪整理的用户。

## 10.3 混合模式

默认模式。

上半部分是系统档案，下半部分是近期日记记录。

MVP 默认使用混合模式，但保留切换按钮。

# 11. 页面气质要求

## 首页

感觉：

```text
进入系统
继续任务
被接住
但不闲聊
```

首页不能只是空聊天框。  
必须有控制台感。

## 引导式理解页

感觉：

```text
系统正在理解我
一步一步
不是审问
```

必须显示当前阶段。

## 计划页

感觉：

```text
裁决
取舍
现实
行动
```

计划页不是待办列表。  
它是 XUANOS 的判断结果。

## 行动反馈页

感觉：

```text
轻量
快速
真实结果
修正判断
```

反馈后必须显示系统如何修改判断。

## 我的系统页

感觉：

```text
个人档案
私人日记
成长系统
被持续理解
可纠正
```

这是 XUANOS 的灵魂页。

# 12. 动效原则

MVP 允许少量高级动效。

允许：

- 粒子缓慢漂浮；
- 卡片轻微淡入；
- 按钮 hover 微光；
- 当前阶段高亮；
- 反馈提交后系统修正卡片出现；
- 金色细线缓慢扫过；
- Apple-like glass 的轻微高光变化。

禁止：

- 复杂 3D；
- 过度扫描线；
- 大量发光文字；
- 游戏化爆炸动效；
- 影响阅读的动态背景；
- 页面过度闪烁。

动效只服务系统感，不服务炫技。

---

# 13. MVP 视觉验收标准

阶段 1 静态原型必须满足：

1. 第一眼不像普通 AI 聊天网页；
2. 黑金风格统一；
3. 粒子和毛玻璃体现系统感；
4. Apple-like glass 清晰、克制、不影响阅读；
5. 东方极简明显，但不玄学；
6. 卡片层级清楚；
7. 文字短句化、状态化；
8. 计划页有取舍和裁决感；
9. 我的系统页可在档案感和日记感之间切换；
10. 行动反馈页足够轻；
11. 用户知道可以修改系统判断。
